# 123

## Instructions

Действуй как Senior Java-разработчик и эксперт по Paper API. >
Моя задача: написать продвинутый плагин (или модуль) на Систему Скиллов (Навыков) для сервера Minecraft.

Технический стек и стандарты:

Ядро и язык: Paper 1.21.11, Java 21.

База данных: SQLite / MySQL через пул соединений HikariCP. Все SQL-запросы (сохранение, загрузка) должны выполняться строго асинхронно (например, через CompletableFuture), чтобы не фризить основной поток (Main Thread).

Текст и локализация: Использовать только MiniMessage (Adventure API) для всего текста (сообщения, названия меню, лор). Никаких устаревших § или &. Все строки должны выноситься в lang.yml.

Кэширование: Использовать потокобезопасные коллекции (например, ConcurrentHashMap) или библиотеку Caffeine для хранения данных игроков в памяти, пока они онлайн. Сохранять данные при PlayerQuitEvent и регулярно сбрасывать в БД.

GUI: Меню со скиллами должно быть интерактивным, защищенным от дюпов (отмена кликов/shift-кликов) и иметь динамический размер.

Требования к функционалу Скиллов:

Типы навыков: Реализовать базовую архитектуру для добавления различных навыков (например, Шахтер, Фермер, Охотник).

Прогрессия: Система получения опыта (EXP) за определенные действия (сломал блок, убил моба). У каждого уровня должен быть свой порог EXP (формула или конфиг).

Награды/Бафы: При достижении новых уровней игрок должен получать пассивные бафы (например, шанс двойного дропа, эффекты зелий) или открывать новые возможности.

Слушатели (Listeners): Эффективная обработка эвентов (BlockBreakEvent, EntityDeathEvent) без создания лагов.
